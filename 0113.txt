整数型五类：

tinyint 迷你整形型，一个字节，最多表示256种（8位）
smallint 小整形，两个字节，表示65536种（16位）
mediumint  三个字节（24位）
int / integer 四个字节（32位）
bigint 八个字节（64位）


注意： 数值类型都是默认有符号的，符号限定：int unsigned;

查看表结构（desc 表名）时，数据类型后的括号中的数字是表示显示宽度，例如， -123 是四位宽度，224是三位宽度，此宽度可控，但不会改变数据本身的大小，其作用在于当数据不够显示宽度时，会自动让数据变成对应的显示宽度，通常需要搭配一个前导0来增加宽度，不改变值大小，即  zerofill，其意义在于保证数据格式的整齐性



小数型

两种：浮点型（小数点浮动精度有限可能丢失精度）
      定点型（小数点固定，精度固定，不会丢失精度）


浮点型：四舍五入
       float 单精度，4个字节存储，精度为7位左右
       double 双精度，8个字节存储，精度范围15位左右
使用： float(M,D) -- M为总长度，D为小数长度，整数长M-D

插入数据可以是小数也可以是科学计数法
浮点数如果是因为系统的四舍五入而进位，系统是允许的


定点型(decimal)，绝对保证整数部分不会四舍五入，理论上小数部分也不会，但实际可能四舍五入,所以整数部分一定不能超出长度，小数部分可以超





时间日期类型

datetime 时间日期，格式YYYY-mm-dd HH:ii:ss  范围从1000到9999年，，有0值  0000-00-00 00:00:00
date 就是上面的date部分
time 时间（段）  ，-时间到+时间
timestamp 时间戳，从1970年开始的 YYYY-mm-dd HH:ii:ss
year 年份，有year(2)和year(4)两种形式



字符串类型
6类，char varchar text blob enum set


（1）定长字符串
格式：char(L)   单位为字符，最大长度为255
例 ： char(4)  在UTF8 下，需要 4*3=12个字节

（2）变长字符串
varchar  在分配空间时按照最大的分配，但实际占用多少根据具体的数据来确定

varchar(L)  理论最大长度是65536个字符，但是会多出一到二个字节来确定存储的实际长度(实际长度超过255，会使用文本 text 来存储)
例： varchar(10) 若实际存了10个汉字，utf8环境下，10*3+1=31个字节
                 若实际只存了3个汉字，则3*3+1=10个字节

如何选择定长还是变长字符串？
（1）定长的磁盘空间比较浪费，但是效率高，若果数据基本上长度一致，则使用定长，如身份证，电话号码等
（2）变长的磁盘空间比较节省，但是效率低，若数据长度不一致，如姓名，地址等

（3）文本字符串   text / blob

text 存储文字（通常二进制数据通常存储路径）
blob  存储二进制数据，通常不用

（4）枚举 
格式 ：enum(元素列表) 
作用：规范数据格式；
      节省存储空间（枚举别名单选框）：系统自动转换格式，实际存储的是数值而不是字符串本身
证明是数值：显示      字段名+0  看是否为数值

枚举原理：枚举在定义时系统自动建立一个数字与枚举元素的对应关系放入日志中，插入数据时系统自动转换成数值存储，而取数据时自动转换成字符串输出显示

直接数值插入枚举元素，也可以插入成功（0113.SQL文档）




（5）集合字符串  
实际存储的是数值，而不是字符串（集合是多选）
set（元素列表）

集合能够规范数据和节省空间





MySQL记录长度
MySQL规定任何一条记录的最大长度不能超过65535个字节（varchar永远达不到理论值）

varchar 实际长度多长？要看字符集编码
UTF8下：varchar 实际长度 21845（字符） *3  = 65535（字节） ,但还要加上 2 字节长度存储值，为65534字节
gbk下，实际32767（字符） * 2 + 2 = 65534（字节）

MySQL中text文本字符串不占记录长度，额外存储，但是text文本字符串也是属于记录的一部分，一定需要占据记录中的部分长度：10个字节（用来保存数据的地址和长度）




列属性
   真正约束字段的是数据类型，但是比较单一，所以需要额外约束来保证合法性
列属性包括：NULL、NOT NULL、default、primary key ,auto_increment,comment等

空属性：  NULL（默认） 和 NOT NULL
空值无法参与运算，所以实际尽量不使用空值，实际案例（看SQL文档）



列描述 
comment  描述，没有实际含义：是专门用来描述字段，会根据表创建语句保存，用来给程序员（DBA）来进行了解的


默认值：
某种数据经常性的出现某个具体的值，可以在一开始就指定好，需要修改时再修改
关键字  ： default

默认值的生效：插入数据时不给该字段赋值，则使用的是默认值

想要使用默认值，可以使用default关键字代替值
